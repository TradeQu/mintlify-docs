## What fired

Database query latency at the p95 percentile crossed its threshold.

| Alert | Condition | For |
| --- | --- | --- |
| `MeshQuDatabaseSlowQueries` (warning) | p95 query duration > **100ms** | 5 min |

## Symptoms

- Slow database queries surface as elevated API request latency — cross-check [high-latency](/runbooks/high-latency).
- The DB query-duration panel is above the threshold line.
- Evaluate / record calls that touch persistence are slow.

## First actions

1. **Confirm sustained vs transient.** A migration, a backup, or a one-off heavy query can spike p95 briefly.
2. **Check load.** Is query volume unusually high, or are individual queries slow? Distinguish a throughput problem from a per-query problem.
3. **Check the database health.** Connection saturation, CPU, and lock contention are the usual culprits for broad slowdowns.
4. **Correlate with deploys.** A recent change to a query path or an index can regress query timing. Roll back if onset lines up.
5. **Escalate to platform.** If the database itself is degraded (provider-side), engage the database/platform owner per the internal escalation path.
